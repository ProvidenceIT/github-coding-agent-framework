# LeesWijs - AI-Powered Personalized Bible Reading Plan Generator

## Project Overview

LeesWijs is an AI-powered web application that generates personalized Bible reading plans based on users' life situations, interests, experience level, and available time. The app includes daily readings with reflection questions, progress tracking with streaks, and reminder notifications.

### Core Value Proposition
- Personalized Bible reading plans (not one-size-fits-all)
- AI-generated plans that adapt to user's life situation
- Progress tracking with motivating streak system
- Daily email reminders at user-preferred times
- PWA for offline access and mobile-first experience

### Target Users
- New believers looking for guidance on where to start
- Returning Christians wanting to re-engage with Bible reading
- Busy professionals needing short, relevant daily readings
- Young adults seeking accessible Bible content
- Bible study groups wanting shared reading plans

## Database Schema

### Feature 1: User Management Schema
Create PostgreSQL schema for users table with the following fields:
- id (UUID, primary key, auto-generated)
- email (VARCHAR 255, unique, not null)
- name (VARCHAR 255)
- password_hash (VARCHAR 255, nullable for magic link users)
- preferred_translation (VARCHAR 10, default 'HSV')
- reminder_time (TIME, default '08:00')
- reminder_enabled (BOOLEAN, default true)
- timezone (VARCHAR 50, default 'Europe/Amsterdam')
- subscription_tier (VARCHAR 20, default 'free')
- subscription_expires_at (TIMESTAMP, nullable)
- created_at (TIMESTAMP, default NOW())
- updated_at (TIMESTAMP, default NOW())

### Feature 2: Intake Responses Schema
Create intake_responses table to store user questionnaire answers:
- id (UUID, primary key)
- user_id (UUID, foreign key to users, cascade delete)
- bible_experience (VARCHAR 50: 'beginner', 'some', 'regular', 'advanced')
- life_situation (JSONB array of selected situations)
- interests (JSONB array of selected topics)
- daily_minutes (INTEGER: 5, 10, 15, 20, 30)
- duration (VARCHAR 20: '1week', '2weeks', '1month', '3months', '6months', '1year')
- include_ot (BOOLEAN, default true)
- include_nt (BOOLEAN, default true)
- created_at (TIMESTAMP)

### Feature 3: Reading Plans Schema
Create reading_plans table:
- id (UUID, primary key)
- user_id (UUID, foreign key to users)
- title (VARCHAR 255, not null)
- description (TEXT)
- total_days (INTEGER, not null)
- start_date (DATE)
- status (VARCHAR 20: 'active', 'paused', 'completed', 'abandoned')
- current_day (INTEGER, default 1)
- ai_generated (BOOLEAN, default true)
- created_at (TIMESTAMP)
- completed_at (TIMESTAMP, nullable)

### Feature 4: Plan Days Schema
Create plan_days table for individual reading assignments:
- id (UUID, primary key)
- plan_id (UUID, foreign key to reading_plans, cascade delete)
- day_number (INTEGER, not null)
- title (VARCHAR 255)
- passage (VARCHAR 100, not null) - e.g., "Johannes 1:1-18"
- estimated_minutes (INTEGER)
- theme (VARCHAR 255)
- reflection_question (TEXT)
- connection_to_situation (TEXT, nullable)
- UNIQUE constraint on (plan_id, day_number)

### Feature 5: Reading Progress Schema
Create reading_progress table:
- id (UUID, primary key)
- user_id (UUID, foreign key to users)
- plan_day_id (UUID, foreign key to plan_days)
- completed (BOOLEAN, default false)
- completed_at (TIMESTAMP, nullable)
- journal_entry (TEXT, nullable)
- rating (INTEGER, CHECK 1-5, nullable)
- UNIQUE constraint on (user_id, plan_day_id)

### Feature 6: Streaks Schema
Create streaks table for gamification:
- id (UUID, primary key)
- user_id (UUID, foreign key to users, unique)
- current_streak (INTEGER, default 0)
- longest_streak (INTEGER, default 0)
- last_read_date (DATE)
- updated_at (TIMESTAMP)

### Feature 7: Reminder Log Schema
Create reminder_log table for email tracking:
- id (UUID, primary key)
- user_id (UUID, foreign key to users)
- sent_at (TIMESTAMP, default NOW())
- status (VARCHAR 20: 'sent', 'failed', 'opened')
- plan_id (UUID, foreign key to reading_plans)

### Feature 8: Database Indexes
Create performance indexes:
- idx_reading_plans_user ON reading_plans(user_id)
- idx_reading_plans_status ON reading_plans(status)
- idx_reading_progress_user ON reading_progress(user_id)
- idx_streaks_user ON streaks(user_id)
- idx_plan_days_plan ON plan_days(plan_id)
- idx_reminder_log_user ON reminder_log(user_id)

## Backend API (tRPC Routers)

### Feature 9: Auth Router
Create tRPC auth router with procedures:
- sendMagicLink: Send magic link email for passwordless login
- verifyMagicLink: Verify token and create session
- getSession: Get current user session
- logout: Invalidate session
- updateProfile: Update user profile settings

### Feature 10: Intake Router
Create tRPC intake router:
- submitIntake: Save intake questionnaire responses
- getLatestIntake: Retrieve user's most recent intake
- updateIntake: Modify existing intake responses

### Feature 11: Plan Router
Create tRPC plan router:
- generatePlan: Call Claude AI to generate personalized reading plan
- getActivePlan: Get user's current active plan
- getAllPlans: List all user's plans with pagination
- getPlanById: Get specific plan with all days
- updatePlanStatus: Pause, resume, or abandon plan
- deletePlan: Soft delete a plan

### Feature 12: Progress Router
Create tRPC progress router:
- markDayComplete: Mark a reading day as completed
- markDayIncomplete: Unmark a completed day
- addJournalEntry: Add/update journal entry for a day
- addRating: Rate a reading (1-5 stars)
- getTodayReading: Get today's assigned reading
- getWeekProgress: Get progress for current week

### Feature 13: Streak Router
Create tRPC streak router:
- getStreak: Get user's current and longest streak
- updateStreak: Internal procedure to update streak on completion
- getLeaderboard: Optional - get top streaks (anonymized)

### Feature 14: Reminder Router
Create tRPC reminder router:
- updateReminderSettings: Set reminder time and enable/disable
- getReminderSettings: Get current reminder configuration
- testReminder: Send test reminder email

### Feature 15: Subscription Router
Create tRPC subscription router:
- createPayment: Initialize Mollie payment for subscription
- handleWebhook: Process Mollie payment webhooks
- getSubscriptionStatus: Check current subscription tier
- cancelSubscription: Cancel active subscription

## AI Plan Generation

### Feature 16: Claude AI Service
Create AI service for plan generation using Claude 3.5 Sonnet:
- Accept intake data (experience, situation, interests, time, duration)
- Generate structured JSON plan with all days
- Include theme, passage, estimated reading time, reflection question
- Connect passages to user's life situation where relevant
- Handle Dutch language for all generated content

### Feature 17: AI Prompt Template
Create optimized prompt template for plan generation:
- Include user profile context
- Apply guidelines for beginner-friendly progression
- Avoid difficult passages (genealogies, laws) for beginners
- Balance OT/NT based on preferences
- Calculate appropriate passage lengths based on daily minutes
- Output structured JSON format

### Feature 18: Plan Validation
Validate AI-generated plans:
- Verify all Bible references are valid
- Check estimated reading times are reasonable
- Ensure day count matches requested duration
- Validate JSON structure before database insertion

### Feature 19: Plan Regeneration
Allow users to regenerate plans:
- Keep original intake data
- Generate new plan with different passages
- Option to specify what didn't work about previous plan
