## üîê Setup with SSH (Option 2: SSH Keys)

### 1. Add SSH Key to GitHub

```bash
# Generate SSH key if you don't have one
ssh-keygen -t ed25519 -C "your_email@providence.it"

# Add to ssh-agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Copy public key
cat ~/.ssh/id_ed25519.pub
```

**Then:**
1. Go to https://github.com/settings/keys
2. Click "New SSH key"
3. Paste your public key
4. Save

### 2. Update Remote to SSH

```bash
cd /home/bram-dokman/Linear-Coding-Agent-Harness

# Change to SSH URL
git remote set-url origin git@github.com:ProvidenceIT/Linear-Coding-Agent-Harness.git

# Test connection
ssh -T git@github.com

# Push
git push -u origin main
```

## üîß Troubleshooting

### Issue: "Authentication failed"

**Solution 1: GitHub CLI**
```bash
gh auth login
gh auth status
```

**Solution 2: Personal Access Token**
```bash
# Create token at https://github.com/settings/tokens
# Use as password when git push prompts
```

**Solution 3: SSH Keys**
```bash
# Generate and add SSH key (see Option 2 above)
git remote set-url origin git@github.com:ProvidenceIT/Linear-Coding-Agent-Harness.git
```

---

### Issue: "Repository not found"

**Check:**
1. Repository exists: https://github.com/ProvidenceIT/Linear-Coding-Agent-Harness
2. You have access (member of ProvidenceIT organization)
3. Remote URL is correct: `git remote -v`

**Fix:**
```bash
# If repository doesn't exist, create it:
gh repo create ProvidenceIT/Linear-Coding-Agent-Harness --public

# Or create via web interface
```

---

### Issue: "Permission denied"

**Check:**
1. You're a member of ProvidenceIT organization
2. You have write access to the repository
3. Organization settings allow member repository creation

**Fix:**
Ask organization admin to:
1. Add you as member: https://github.com/orgs/ProvidenceIT/people
2. Grant repository permissions
3. Or create repository and add you as collaborator

---
